# AstraLinux DevOps Portfolio

## О проекте

Данный репозиторий содержит выпускную квалификационную работу по администрированию операционной системы Astra Linux.

Проект представляет собой комплексную практическую реализацию задач системного администрирования и базовых DevOps-подходов: от настройки рабочего места пользователя до развёртывания корпоративного сервера с сетевыми сервисами и контейнеризацией.

---

## Цели проекта

* Развертывание и настройка пользовательской рабочей станции
* Организация отказоустойчивого хранилища данных (RAID 5)
* Управление пользователями и правами доступа
* Настройка дисковых квот
* Конфигурация серверной инфраструктуры
* Настройка сетевого взаимодействия
* Развертывание контейнеров
* Анализ сетевого трафика
* Подготовка инфраструктуры к автоматизации (Ansible)

## Реализованный функционал

### Рабочая станция

* Установка и базовая настройка системы
* Создание RAID 5 массива с использованием `mdadm`
* Форматирование и монтирование файловой системы (ext3)
* Настройка автоматического монтирования при загрузке
* Создание пользователей и групп
* Управление правами доступа
* Настройка дисковых квот (user/group quotas)
* Настройка sudo-доступа
* Мониторинг системы (`htop`)

---

### Серверная часть

* Развертывание серверной системы
* Настройка сетевых интерфейсов (внешняя и внутренняя сети)
* Конфигурация IP-адресации и маршрутизации
* Включение IP forwarding
* Настройка DNS-сервера
* Проверка сетевого взаимодействия

---

### Контейнеризация и DevOps-практики

* Установка и настройка Docker
* Запуск контейнеров (`hello-world`, `busybox`)
* Создание собственного Docker-образа
* Развертывание веб-сервиса в контейнере
* Управление контейнерами и образами

---

### Анализ и диагностика

* Использование `tcpdump` для анализа DNS-трафика
* Проверка сетевых пакетов
* Базовая диагностика работы сервисов
